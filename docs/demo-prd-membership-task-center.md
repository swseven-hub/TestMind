# 会员任务中心 PRD（演示版）

## 1. 文档信息

| 项目 | 内容 |
| --- | --- |
| 产品名称 | 会员任务中心 |
| 文档版本 | V1.0 Demo |
| 适用端 | Web / App |
| 目标用户 | 普通会员、高级会员、运营管理员 |
| 文档用途 | 用于演示 AI 测试用例生成工具 |

## 2. 背景与目标

当前产品希望通过任务激励提升用户活跃度。用户可以在任务中心查看任务、领取任务、完成任务并领取积分或优惠券奖励。

本期目标：

1. 支持用户登录后查看任务中心。
2. 支持按任务状态和任务类型筛选任务。
3. 支持用户领取任务、查看任务进度、完成任务。
4. 支持任务完成后领取积分或优惠券奖励。
5. 支持不同会员等级展示不同任务。
6. 支持风控命中用户限制领取高价值任务。

## 3. 术语说明

| 术语 | 说明 |
| --- | --- |
| 任务 | 运营配置的用户行为目标，例如签到、浏览商品、完成购买 |
| 任务状态 | 可领取、进行中、待领奖、已领奖、已过期 |
| 积分 | 用户账户内的虚拟奖励，可累积 |
| 优惠券 | 用户领取后进入券包，可在下单时使用 |
| 风控命中 | 用户触发风险规则，例如异常设备、频繁领取、高风险账号 |

## 4. 用户角色

| 角色 | 权限 |
| --- | --- |
| 未登录用户 | 可看到登录入口，不可领取任务和领取奖励 |
| 普通会员 | 可查看和领取普通任务 |
| 高级会员 | 可查看和领取普通任务、高级会员专属任务 |
| 风控命中用户 | 可查看任务，但不可领取高价值任务 |
| 运营管理员 | 在后台配置任务、奖励、库存、有效期 |

## 5. 功能范围

### 5.1 登录与注册

#### 功能说明

用户进入任务中心前需要登录。支持手机号密码登录和手机号验证码注册。

#### 业务规则

1. 已注册用户输入正确手机号和密码后可登录成功。
2. 密码错误时登录失败，并提示“手机号或密码错误”。
3. 未注册手机号登录时，引导用户注册。
4. 验证码注册时，手机号格式必须合法，验证码必须正确。
5. 用户注册时必须勾选用户协议。
6. 账号被锁定时，即使密码正确也不允许登录。
7. 登录按钮提交中需要置灰，避免重复点击产生多次登录请求。
8. 登录接口超时或失败时，需要提示用户重试。

#### 字段规则

| 字段 | 规则 |
| --- | --- |
| 手机号 | 必填，11 位数字，以 1 开头 |
| 密码 | 必填，8-20 位，支持字母、数字和符号 |
| 验证码 | 必填，6 位数字，5 分钟内有效 |

### 5.2 任务列表

#### 功能说明

用户登录后进入任务中心，默认展示可参与任务列表。

#### 业务规则

1. 任务卡片展示任务名称、任务类型、奖励内容、截止时间、当前状态和操作按钮。
2. 默认排序规则：推荐任务优先，其次按截止时间升序。
3. 用户可以按任务状态筛选：全部、可领取、进行中、待领奖、已完成、已过期。
4. 用户可以按任务类型筛选：签到、浏览、购买、分享。
5. 筛选结果为空时展示空状态，并提供“清除筛选”入口。
6. 列表加载失败时展示失败状态，并提供“重试”入口。
7. 任务标题过长时需要换行或省略，不允许遮挡奖励和按钮。
8. 当历史任务超过 50 条时，需要分页加载。

### 5.3 任务领取与进度

#### 功能说明

用户可以领取符合条件的任务，领取后任务状态变为进行中，并显示完成进度。

#### 业务规则

1. 只有可领取状态的任务可以领取。
2. 已领取任务再次点击领取，不允许重复创建任务实例。
3. 任务过期后不允许继续领取或完成。
4. 任务进度达到完成条件后，状态变为待领奖。
5. 未达到完成条件时，不允许领取奖励。
6. 任务完成条件按任务类型判断：
   - 签到任务：当天签到 1 次。
   - 浏览任务：浏览指定页面不少于 10 秒。
   - 购买任务：完成一笔实付金额大于 0 的订单。
   - 分享任务：成功分享并回流访问 1 次。
7. 任务进度同步失败时，页面需要提示“进度同步中，请稍后刷新”。

#### 状态流转

```text
可领取 -> 进行中 -> 待领奖 -> 已领奖
可领取 -> 已过期
进行中 -> 已过期
```

### 5.4 奖励领取

#### 功能说明

任务完成后，用户可以领取积分或优惠券奖励。

#### 业务规则

1. 任务状态为待领奖时，显示“领取奖励”按钮。
2. 奖励领取成功后，任务状态变为已领奖。
3. 积分奖励领取成功后，用户积分余额增加，并生成积分流水。
4. 优惠券奖励领取成功后，优惠券进入用户券包。
5. 重复点击领取奖励按钮，只允许发放一次奖励。
6. 奖励库存不足时，领取失败，并提示“奖励已领完”。
7. 奖励服务返回成功但前端超时时，刷新后应展示最终一致状态，不允许重复发奖。
8. 未登录用户或登录态过期用户不允许领取奖励。

#### 奖励配置规则

| 奖励类型 | 规则 |
| --- | --- |
| 积分 | 最小 1 分，最大 1000 分 |
| 优惠券 | 必须配置面额、有效期、使用门槛 |
| 库存 | 库存为 0 时不允许领取 |

### 5.5 会员等级与风控

#### 功能说明

不同会员等级可参与不同任务。高风险用户不能领取高价值任务。

#### 业务规则

1. 普通会员只能看到普通任务。
2. 高级会员可以看到普通任务和高级会员专属任务。
3. 用户会员等级提升后，任务列表需要刷新可参与任务。
4. 任务等级门槛为“大于等于”，例如等级 3 用户可以领取等级门槛为 3 的任务。
5. 普通会员不能通过篡改任务 ID 领取高级任务。
6. 风控命中用户领取高价值任务时，服务端必须拒绝请求。
7. 风控服务超时时，默认拒绝高价值任务领取，并提示稍后重试。

## 6. 异常场景

1. 网络断开时，页面展示网络异常提示。
2. 登录态过期时，跳转登录页或弹出登录弹窗。
3. 接口返回 5xx 时，展示失败提示和重试入口。
4. 任务不存在或已下线时，详情页展示任务不可用。
5. 奖励发放中断时，需要通过奖励流水查询最终状态。

## 7. 权限与安全

1. 所有领取任务和领取奖励接口必须校验登录态。
2. 服务端必须校验会员等级，不能只依赖前端按钮隐藏。
3. 服务端必须校验任务状态、任务有效期、奖励库存。
4. 同一用户同一任务只能领取一次奖励。
5. 关键操作需要记录操作日志：领取任务、完成任务、领取奖励、发奖失败。

## 8. 性能要求

1. 任务中心首页首屏接口响应时间 P95 小于 1 秒。
2. 任务列表分页每页 20 条，滚动加载不能出现明显卡顿。
3. 领取任务接口响应时间 P95 小于 800 毫秒。
4. 领取奖励接口响应时间 P95 小于 1.5 秒。
5. 高频点击按钮时，不允许产生重复领取或重复发奖。

## 9. 验收标准

1. 用户可以完成登录、查看任务、领取任务、完成任务、领取奖励的完整流程。
2. 登录失败、未注册、账号锁定、验证码错误等场景均有明确提示。
3. 任务列表支持状态和类型筛选，空状态和失败状态展示正确。
4. 任务领取、完成、领奖状态流转正确，不允许重复领取或重复发奖。
5. 普通会员、高级会员、风控命中用户的可见任务和可操作范围符合规则。
6. 接口超时、失败、库存不足、任务过期等异常场景均能正确处理。
