# 登录功能 PRD（极简演示版）

## 1. 文档信息

| 项目 | 内容 |
| --- | --- |
| 产品名称 | 示例 App |
| 功能模块 | 登录 |
| 文档版本 | V1.0 Demo |
| 适用端 | iOS / Android |
| 文档用途 | 用于演示 AI 测试用例生成工具 |

## 2. 背景与目标

用户进入 App 后，需要通过手机号和密码登录。登录成功后进入首页，登录失败时需要明确提示原因，并保证用户不会因为重复点击、网络异常或账号异常进入错误状态。

本期只覆盖手机号密码登录，不包含注册、验证码登录、第三方登录和找回密码。

## 3. 用户角色

| 角色 | 权限 |
| --- | --- |
| 未登录用户 | 可以进入登录页并提交登录 |
| 已登录用户 | 再次打开 App 时直接进入首页 |
| 锁定账号用户 | 不允许登录，需要提示账号被锁定 |

## 4. 功能范围

### 4.1 登录页展示

1. 登录页展示手机号输入框、密码输入框、显示/隐藏密码按钮、登录按钮。
2. 手机号和密码为空时，登录按钮置灰不可点击。
3. 用户输入手机号和密码后，登录按钮变为可点击。
4. 密码默认隐藏，点击显示密码按钮后可明文展示，再次点击恢复隐藏。

### 4.2 手机号密码登录

1. 用户输入已注册手机号和正确密码，点击登录后进入首页。
2. 登录成功后，本地保存登录态。
3. 已登录用户重新打开 App 时，直接进入首页，不再展示登录页。
4. 用户点击退出登录后，清除本地登录态，并回到登录页。

### 4.3 登录失败处理

1. 手机号格式错误时，提示“请输入正确的手机号”。
2. 密码为空时，提示“请输入密码”。
3. 手机号未注册时，提示“账号不存在”。
4. 密码错误时，提示“手机号或密码错误”。
5. 账号被锁定时，提示“账号已被锁定，请联系客服”。
6. 登录接口超时时，提示“网络异常，请稍后重试”。
7. 登录接口返回服务异常时，提示“登录失败，请稍后重试”。

### 4.4 重复提交与加载状态

1. 登录请求提交后，登录按钮展示加载状态。
2. 登录请求未完成前，登录按钮不可重复点击。
3. 登录请求结束后，按钮恢复正常状态。
4. 用户在登录请求过程中切换到后台再返回，不应重复发起登录请求。

## 5. 字段规则

| 字段 | 规则 |
| --- | --- |
| 手机号 | 必填，11 位数字，以 1 开头 |
| 密码 | 必填，8-20 位，支持字母、数字和符号 |

## 6. 性能要求

1. 登录接口正常情况下 2 秒内返回结果。
2. 弱网环境下登录页不能卡死，必须保留错误提示和重试能力。

## 7. 非功能要求

1. 密码输入内容不能明文写入日志。
2. 登录态过期后，用户再次打开 App 需要回到登录页。
3. 不允许未登录用户通过接口或页面跳转直接访问首页数据。
